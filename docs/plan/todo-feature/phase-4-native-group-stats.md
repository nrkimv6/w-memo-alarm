## 1. Native Fullscreen Alert (Android)

> PRD 참조: 8.3 Native App 상세, 8.4 플랫폼별 전략, 8.6 Native Fullscreen Alert UI

### 1-1. Capacitor 커스텀 플러그인 개발

- [ ] Android 네이티브 플러그인 프로젝트 셋업
  - Capacitor 커스텀 플러그인 구조 생성
  - Java/Kotlin으로 FullscreenAlarmActivity 구현
  - PRD 8.3의 "구현 방법 (Android)" 참고

- [ ] FullscreenAlarmActivity 구현
  - 잠금화면 위에 표시 (setShowWhenLocked, setTurnScreenOn)
  - 키가드 해제 (requestDismissKeyguard)
  - 전체화면 UI 레이아웃
  - PRD 8.6의 "Android: FullscreenAlarmActivity" 레이아웃 참고

- [ ] 플러그인 ↔ JS 브릿지
  - Capacitor 브릿지로 JS에서 fullscreen alert 트리거 가능하게
  - 메서드: showFullscreenAlert(todoId, title, dueDate)
  - 콜백: onComplete, onPostpone, onDismiss

### 1-2. Fullscreen Alert UI

- [ ] 전체화면 레이아웃 구현 (Android Native XML)
  - 벨 아이콘 + "지금 해야 해요!" 텍스트
  - 할일 제목 + 기한 표시
  - [완료 ✓] [미루기] [닫기] 버튼
  - PRD 8.6의 UI 참고

- [ ] 완료/미루기/닫기 액션 처리
  - 완료 → todoStatus='completed' 변경 + Activity 종료
  - 미루기 → 미루기 바텀시트 열기 (앱 내에서) + Activity 종료
  - 닫기 → Activity 종료만

### 1-3. Alert 채널 분리

- [ ] Android Notification Channel 생성
  - "할일 알람" 채널: 높은 우선순위, 커스텀 소리, 진동
  - "할일 상기" 채널: 기본 우선순위
  - 기존 알림 채널과 분리

### 1-4. Fullscreen Alert 트리거 연동

- [ ] 기존 Phase 2의 알람 발송 로직에서 Native 환경 분기
  - Capacitor 환경이면 Fullscreen Alert 사용
  - PWA 환경이면 기존 requireInteraction 푸시 유지
  - PRD 8.4의 플랫폼별 전략 표 참고

## 7. 대량 처리 (Batch Actions)

> PRD 참조: 10.2 항목 2 — 대량 처리

### 7-1. 다중 선택 모드

- [ ] 할일 목록에서 다중 선택 모드 진입

  - 체크박스로 여러 할일 선택

  - 선택한 할일 모두 같은 날짜로 미루기

## 8. 할일 ↔ 메모 전환

> PRD 참조: 10.2 항목 6

### 8-1. 메모 → 할일 전환

- [ ] 일반 메모를 할일로 전환하는 기능
  - 메모 상세 > 더보기 메뉴 > "할일로 전환"
  - memoType = 'todo', todoStatus = 'pending' 설정
  - 기존 내용/태그 유지

### 8-2. 할일 → 메모 전환

- [ ] 할일 완료 후 메모로 보관하는 기능
  - 할일 상세 > 더보기 메뉴 > "메모로 전환"
  - memoType = 'note'로 변경
  - 완료 기록은 유지 (참고용)

### 8-3. 체크리스트 → 할일 마이그레이션

- [ ] 기존 checklist가 있는 메모를 할일로 전환 안내
  - 체크리스트 항목을 개별 할일로 변환하는 기능
  - 선택적 기능 (사용자 요청 시)

## 완료 기준

Phase 4가 "완료"되려면 아래 시나리오가 모두 동작해야 한다:

1. Android Native에서 잠금화면 위에 전체화면 알람이 표시된다
2. Fullscreen 알람에서 완료/미루기/닫기가 동작한다
3. 알람 채널이 분리되어 별도 소리/진동 설정이 가능하다
4. 배터리 최적화 해제 안내가 표시된다
5. 태그/폴더로 할일을 그룹핑하여 볼 수 있다
6. 일간/주간/월간 완료 통계가 표시된다
7. 여러 할일을 선택하여 일괄 완료/미루기/삭제할 수 있다
8. 메모 ↔ 할일 전환이 가능하다

**Phase 4에서 동작하지 않는 것 (Phase 5 이후):**
- 캘린더 뷰
- 자연어 입력
- 서브태스크
- Android 위젯
- 알림 피로도 관리
- On-Device AI 연동
