## 2. 인스턴스 자동 생성 (JIT 방식)

> PRD 참조: 7.3 반복 할일의 인스턴스 생성 시점

### 2-1. 첫 번째 인스턴스 생성

- [ ] 반복 할일 최초 생성 시 첫 인스턴스 1개 자동 생성
  - 가장 가까운 다음 일정 날짜 계산
  - todoInstances = [{ id, scheduledDate, status: 'pending', postponeCount: 0 }]
  - PRD 7.3 시나리오 1 참고

### 2-2. 완료 시 다음 인스턴스 생성

- [ ] 현재 인스턴스 완료(completed) 처리 시
  - 현재 인스턴스: status = 'completed', completedAt = now
  - 다음 인스턴스 1개 자동 생성 (recurrence 패턴 기반)
  - 종료 조건(endDate, endAfter) 확인 — 종료이면 인스턴스 미생성
  - PRD 7.3 시나리오 2 참고

### 2-3. 건너뛰기 시 다음 인스턴스 생성

- [ ] 현재 인스턴스 건너뛰기(skipped) 처리 시
  - 현재 인스턴스: status = 'skipped', skippedAt = now, skipReason = 입력값
  - 다음 인스턴스 1개 자동 생성
  - PRD 7.3 시나리오 3 참고

### 2-4. 미루기 시 인스턴스 처리

- [ ] 미루기는 새 인스턴스를 생성하지 않음
  - 현재 인스턴스의 scheduledDate만 변경
  - postponeCount 증가
  - 다음 정규 일정은 영향 없음
  - PRD 7.3 시나리오 4 참고

### 2-5. 다음 일정 날짜 계산 함수

- [ ] getNextOccurrence(recurrence, fromDate) 유틸 함수 구현
  - daily: fromDate + interval일
  - weekly: 다음 해당 요일 (daysOfWeek 기준)
  - monthly: 다음 해당 일자 (dayOfMonth 기준)
  - custom: interval 기반 계산
  - 여러 요일이 선택된 경우 가장 가까운 다음 요일

### 2-6. 앱 시작 시 누락 인스턴스 복구

- [ ] 앱 시작 시 반복 할일 체크
  - 활성 인스턴스(status='pending')가 없고
  - 마지막 인스턴스가 completed/skipped인 경우
  - 다음 인스턴스 자동 생성
  - 오프라인 장기 사용 후 앱 열 때 필요
  - PRD 7.3 시나리오 5 참고

- [ ] **[결정 필요]** 활성 인스턴스가 overdue 상태일 때의 처리
  - 자동 skip 후 다음 인스턴스 생성?
  - 사용자가 처리할 때까지 대기?

## 5. 할일 모아보기 — 일정 기준 표시

> PRD 참조: 9.1 핵심 개념, 9.2 일정 기준 정렬, 9.3 반복 일정 계산

### 5-1. 미래 일정 계산 함수

- [ ] getUpcomingSchedules(memo, count) 함수 구현
  - 반복 패턴에서 다음 N개 일정을 계산
  - 실제 인스턴스를 생성하지 않고 SchedulePreview 반환
  - PRD 9.3의 함수 시그니처 및 로직 참고

- [ ] SchedulePreview 타입 정의
  - date, isPostponed, isActive 필드
  - isActive = true: 현재 활성 인스턴스 (실제 DB 데이터)
  - isActive = false: 미래 일정 (계산된 데이터)

### 5-2. 모아보기 데이터 소스 통합

- [ ] 실제 데이터 (DB) + 계산된 데이터 (메모리) 합산

  - 반복 할일의 미래 일정 → getUpcomingSchedules에서 계산
  - 미래 일정은 연한 색상 또는 점선 스타일로 구분
  - PRD 7.3의 "일정별 통합 조회 시 데이터 소스 구분" 다이어그램 참고

### 5-3. 모아보기 뷰 날짜별 정렬 업데이트

- [ ] "이번주" 탭에서 반복 할일 다음 일정 표시
  - 현재 활성 인스턴스를 해당 날짜에 배치
  - PRD 9.2의 뷰 레이아웃 참고

  - 미래 일정은 미루기/건너뛰기 불가 표시 (아직 미생성이므로)
  - PRD 9.1의 "전체 탭에서는 미래 일정도 연하게 표시" 참고

## 완료 기준

Phase 3이 "완료"되려면 아래 시나리오가 모두 동작해야 한다:

1. 할일에 반복 패턴(매일/매주/매월/사용자지정)을 설정할 수 있다
2. 반복 종료 조건 (날짜/횟수/무한)을 설정할 수 있다
3. 현재 인스턴스 완료 시 다음 인스턴스가 자동 생성된다
4. 건너뛰기 시 사유를 입력하고 다음 인스턴스가 자동 생성된다
5. 반복 할일의 과거 인스턴스 이력이 표시된다
6. 할일 모아보기에서 다음 반복 일정 기준으로 표시된다
7. "전체" 탭에서 미래 일정이 연하게 표시된다
8. 미래 일정에는 미루기/건너뛰기가 불가능하다
9. 앱 재시작 시 누락된 인스턴스가 자동 복구된다
10. 반복 할일의 알림이 인스턴스별로 정상 발송된다
