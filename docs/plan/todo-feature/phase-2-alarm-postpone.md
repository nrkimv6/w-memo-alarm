## 1. 상기 알림 (Remind) 발송

> PRD 참조: 3.1.1 1단계 상기, 3.1.2 설정 레이어, 3.1.3 시간축 예시

### 1-1. 상기 알림 스케줄링

- [ ] 매일 설정 시각에 상기 알림을 발송하는 로직 구현
  - 앱 설정의 remind.time (기본 07:00)에 매일 실행
  - Service Worker에서 매일 체크하는 방식 또는 알림 스케줄링
  - 기존 `src/service-worker.ts`의 알림 체크 로직 참고하여 확장

- [ ] Capacitor Local Notifications으로 상기 알림 스케줄
  - `src/lib/utils/capacitor.ts`의 기존 알림 스케줄링 패턴 참고
  - 매일 반복 스케줄로 등록

- [ ] FCM으로 상기 알림 발송 (서버 트리거)
  - 기존 FCM 발송 인프라 활용

### 1-2. 상기 알림 내용 생성

- [ ] 할일 요약 수집 로직 구현
  - **[결정 필요]** 포함 범위 결정 후 구현

- [ ] 알림 메시지 포맷

  - 본문: "미완료: N건 | 기한 오늘: N건"
  - PRD 3.1.2의 "상기시간 ON일 때 알림 내용" 참고

### 1-3. Todo별 개별 상기 설정 연동

- [ ] useGlobalRemind = true인 할일은 앱 설정 시각 사용

- [ ] 개별 상기 추가 (0~N개) — todoTiming.remindTimes 배열에서 읽기

## 2. 알람 (Alert) 발송

> PRD 참조: 3.1.1 2단계 알람, 7.2 알림 채널별 동작

### 2-1. 알람 스케줄링

- [ ] 사용자가 직접 지정한 알람 시각에 발송
  - todoTiming.alertTimes 배열에서 읽기
  - 각 알람 시각마다 개별 알림 스케줄 등록
  - PRD 4.1의 TodoAlertEntry 참고

- [ ] 자동알람 (autoAlertBefore) 계산 및 스케줄링

  - 기한이 없는 할일에는 자동알람 적용 안됨
  - useGlobalAutoAlert = true이면 앱 설정값 사용

- [ ] Service Worker에서 알람 체크 및 발송
- [ ] Capacitor에서 알람 스케줄링 (진동+소리 채널)
- [ ] FCM에서 높은 우선순위 푸시 발송

### 2-2. 알람 강조 표현

- [ ] 푸시 알림에 강조 옵션 적용

- [ ] 알람 알림 클릭 시 할일 상세로 이동

  - Service Worker의 notificationclick 이벤트에서 처리

### 2-3. Alert 앱 내 모달 오버레이 (포그라운드)

- [ ] AlertModal 컴포넌트 생성
  - 앱이 열려있을 때 알람 시각이 되면 전체화면 모달 표시
  - 배경 블러 + 반투명 오버레이
  - "🔔 지금 해야 해요!" + 할일 제목 + 기한
  - [완료 ✓] [미루기] [닫기] 버튼
  - PRD 8.5의 Alert 모달 UI 참고

- [ ] 앱 포그라운드 상태에서 알람 시각 도달 감지

## 완료 기준

Phase 2가 "완료"되려면 아래 시나리오가 모두 동작해야 한다:

1. 매일 설정 시각에 "오늘의 할일 요약" 상기 알림이 발송된다
2. 사용자가 설정한 알람 시각에 강조 알림이 발송된다
3. 자동알람 (기한 N시간 전) 이 정상 발송된다
4. 앱이 열려있을 때 알람 시각에 모달 오버레이가 표시된다
5. 미루기 바텀시트에서 빠른 선택 또는 직접 입력으로 기한을 미룰 수 있다
6. 2회째 미루기부터 경고가 표시되고, 횟수 제한을 설정할 수 있다
7. 횟수 제한 초과 시 미루기가 불가능하다
8. 완료 후 3초간 "실행 취소" 토스트가 표시된다
9. Todo별로 상기/알람을 개별 설정할 수 있다
10. 할일 알림이 알림 내역에 기록된다

**Phase 2에서 동작하지 않는 것 (Phase 3 이후):**
- 건너뛰기 기능은 Phase 3 (반복 할일 전용)
- 반복 할일의 알람은 Phase 3
- Native Fullscreen Alert는 Phase 4
