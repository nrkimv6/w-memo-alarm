# MemoCard UI 개선 계획

> **작성일**: 2026-01-23
> **타입**: UI/UX 개선
> **목표**: 제목 가독성 향상 + 버튼 정리 + 모바일 친화적 스와이프 액션

---

## 📊 현황 분석

### 문제점
- **제목 공간 부족**: 제목 옆에 5개 버튼 (핀, 즐겨찾기, 숨기기, 수정, 삭제)이 배치되어 제목이 좁게 표시됨
- **버튼 과다**: 44px × 5개 = 약 220px 차지
- **모바일 UX 미흡**: 호버 기반 UI는 터치 환경에서 작동 불가
- **중요도 혼재**: 자주 사용하는 기능과 드물게 사용하는 기능이 동일한 레벨에 배치

### 일반적인 메모 앱 UX 패턴
- **Apple Notes, Google Keep**: 카드 표면 최소화, 상세 화면에서 편집
- **Notion, Evernote**: 상태 표시(핀/즐겨찾기)만 카드에 표시, 나머지는 메뉴
- **iOS 앱**: 스와이프 액션으로 빠른 동작 (삭제, 고정 등)

---

## 🎯 개선 목표

1. **제목 공간 최대 확보**: 사용자가 제목을 명확히 인식할 수 있도록
2. **버튼 정리**: 핀만 카드 표면에 노출, 나머지는 케밥 메뉴로 통합
3. **모바일 최적화**: 스와이프 액션 추가 (삭제, 핀 고정)
4. **직관적인 UX**: 첫 실행 시 스와이프 액션 안내

## 📐 개선안

### 1. 카드 레이아웃 변경

#### Before (현재)
```
┌─────────────────────────────────────┐
│ [제목] 📌 ⭐ 👁 ✏️ 🗑️  ← 5개 버튼  │
│                                     │
│ 본문 내용...                        │
└─────────────────────────────────────┘
```

#### After (개선)
```
┌─────────────────────────────────────┐
│ [제목이 넓게 표시됨.............]  │
│                            📌 ⋮     │  ← 2개만
│                                     │
│ 본문 내용...                        │
│                                     │
│ #태그 #태그2                        │
│ ─────────────────────────────────   │
│ 3분 전              📁 폴더명       │
└─────────────────────────────────────┘
```

### 2. 버튼 재배치

| 기능 | Before | After | 이유 |
|------|--------|-------|------|
| **핀 고정** | 카드 표면 | 카드 표면 (유지) | 자주 사용, 시각적 피드백 중요 |
| **즐겨찾기** | 카드 표면 | 상세 화면으로 이동 | 핀과 용도 유사, 우선순위 낮음 |
| **숨기기** | 카드 표면 | 케밥 메뉴 | 드물게 사용 |
| **수정** | 카드 표면 | 케밥 메뉴 | 카드 클릭 → 상세 화면에서도 가능 |
| **삭제** | 카드 표면 | 케밥 메뉴 + 스와이프 | 케밥 또는 왼쪽 스와이프 |

### 3. 케밥 메뉴 (⋮) 구성

```
┌─────────────────┐
│ ✏️  수정         │
│ 🗑️  삭제         │
│ 👁️  숨기기       │
│ 📁 폴더 이동     │
└─────────────────┘
```

### 4. 스와이프 액션 (모바일)

- **왼쪽 스와이프** → 삭제 (빨간색 배경)
- **오른쪽 스와이프** → 핀 고정/해제 (파란색 배경)

### 5. 첫 실행 안내

```
┌─────────────────────────────────────┐
│     💡 메모 카드 사용 팁             │
│                                     │
│  • 왼쪽으로 밀면 삭제 🗑️             │
│  • 오른쪽으로 밀면 고정 📌           │
│  • 메뉴(⋮)를 눌러 더 많은 기능 사용 │
│                                     │
│         [확인]      [다시 보지 않기] │
└─────────────────────────────────────┘
```

## 🛠️ 구현 계획

### Phase 1: 레이아웃 변경 ✅ 준비 완료
- [ ] `MemoCard.svelte` 헤더 영역 리팩토링
  - 제목을 좌측 전체 사용 (`flex-1`)
  - 우측에 핀(📌) + 케밥 메뉴(⋮) 배치
- [ ] 즐겨찾기 버튼 제거 (카드 표면에서)
- [ ] 수정, 삭제, 숨기기 버튼 제거

### Phase 2: 케밥 메뉴 구현
- [ ] `KebabMenu.svelte` 컴포넌트 생성
  - Dropdown 메뉴 (bits-ui 또는 커스텀)
  - 메뉴 항목: 수정, 삭제, 숨기기, 폴더 이동
- [ ] `MemoCard.svelte`에 케밥 메뉴 통합
- [ ] 메뉴 클릭 시 이벤트 핸들러 연결

### Phase 3: 상세 화면 개선
- [ ] `MemoDetailModal.svelte` 확인
  - 즐겨찾기 버튼 추가 (아직 없다면)
  - 수정/삭제 버튼 위치 확인

### Phase 4: 스와이프 액션 구현
- [ ] 스와이프 감지 라이브러리 선택
  - 옵션 1: `svelte-gestures` (가볍고 간단)
  - 옵션 2: Vanilla JS `touchstart/touchmove/touchend`
- [ ] `SwipeableCard.svelte` 래퍼 컴포넌트 생성
  - 왼쪽 스와이프: 삭제 액션 (빨간색 배경 + 🗑️ 아이콘)
  - 오른쪽 스와이프: 핀 고정 액션 (파란색 배경 + 📌 아이콘)
  - 스와이프 임계값: 화면 너비의 30%
- [ ] `MemoCard.svelte`를 `SwipeableCard`로 래핑

### Phase 5: 첫 실행 안내
- [ ] `OnboardingStore.svelte.ts` 확인/수정

- [ ] `SwipeGuideModal.svelte` 모달 생성
  - 스와이프 액션 안내 메시지
  - "확인" / "다시 보지 않기" 버튼
- [ ] 메인 화면에서 첫 실행 시 모달 표시

### Phase 6: 테스트 & 배포
- [ ] 웹 환경 테스트
  - 케밥 메뉴 동작
  - 레이아웃 반응형
